OpenStack Keystone

Identity Service Overview

Welcome everyone! Keystone is a core service for authentication and identity management in OpenStack.

Why Keystone?

  • Before Keystone, OpenStack lacked a unified authentication system.
  • Keystone centralizes identity, token, and role-based access.
Keystone makes OpenStack secure and manageable by consolidating identity and policy control.

Definition

Keystone is OpenStackâ€™s identity service, providing authentication, authorization, and service catalog.

Keystone manages users, tenants (projects), and the services that make up your cloud.

Architecture

  • Identity backend: Stores users/projects
  • Token provider: Issues tokens
  • Catalog: Lists available services
Keystone has a pluggable backend and issues tokens for secure inter-service communication.

How Keystone Works

  1. User provides credentials
  2. Keystone verifies and returns a token
  3. Token is used to access other OpenStack services
Keystone enables secure access to cloud resources using scoped tokens.

Integration

  • Glance, Nova, Neutron: All use Keystone for auth
  • Horizon: Depends on Keystone to log users in
Keystone is essential for any OpenStack component to authenticate and authorize users.

Installation

sudo apt install keystone

Configure /etc/keystone/keystone.conf and sync DB

Keystone setup requires proper integration with the database and configuration of its public endpoint.

Automation with Ansible

- name: Install Keystone
  hosts: keystone
  tasks:
    - name: Install keystone
      apt:
        name: keystone
        state: present
Ansible simplifies deploying Keystone across multiple nodes with reusable playbooks.

Managing Keystone

openstack user list
openstack project create demo
openstack role add --project demo --user demo admin
These commands are commonly used to manage users, roles, and projects in Keystone.

Practice Session

  • Create user and project
  • Assign roles
  • Authenticate and get a token
This practice will reinforce how Keystone controls access and identity.

Importance of Keystone

  • Unified identity and access control
  • Integrates all OpenStack services
  • Enables multi-tenancy and RBAC
Keystone is vital for secure and organized multi-user cloud environments.
