OpenStack Heat

Orchestration Service Overview

Hi everyone! Let’s explore Heat — OpenStack’s orchestration engine for automating cloud resources with templates.

Why Heat?

  • Manual resource management is error-prone and inefficient.
  • Heat automates the deployment and scaling of infrastructure.
Heat reduces operational burden by treating infrastructure as code.

Definition

Heat is the orchestration service in OpenStack, enabling users to define and manage cloud resources using templates (HOT/YAML).

Heat lets you define your entire infrastructure stack declaratively.

Architecture

  • Heat API: Accepts stack templates and operations
  • Heat Engine: Processes templates and manages resources
  • Heat Template (HOT): YAML-based infrastructure-as-code files
Heat’s engine interprets the user template, makes API calls to other services to create resources.

How Heat Works

  1. User submits HOT template
  2. Heat parses and validates template
  3. Resources are created in sequence
Heat coordinates the lifecycle of each resource (e.g., networks, servers, volumes).

Integration

  • NOVA: Launches compute resources
  • Neutron: Creates networks
  • Cinder: Manages volumes
Heat orchestrates other OpenStack services using their public APIs.

Installation

sudo apt install heat-api heat-engine

Edit /etc/heat/heat.conf and register the service in Keystone.

Ensure proper credentials and endpoints are defined in Heat’s config files.

Automation with Ansible

- name: Install Heat
  hosts: heat
  tasks:
    - name: Install packages
      apt:
        name: "{{ item }}"
        state: present
      loop:
        - heat-api
        - heat-engine
Automate Heat's deployment using Ansible for reproducible environments.

Managing Heat

openstack stack create -t template.yaml mystack
openstack stack list
openstack stack delete mystack
These commands manage your infrastructure stacks from the CLI.
